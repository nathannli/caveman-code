# CaveKit Writing (Kit Writing)

Guidelines for authoring CaveKit specification documents (kits) that are clear, complete, and actionable by an AI coding agent.

## Kit Structure

- **Overview**: one-paragraph summary of the feature and its purpose
- **Requirements**: numbered list of functional requirements (R-NNN)
- **Acceptance Criteria**: measurable, binary pass/fail criteria per requirement (AC-NNN)
- **Build Plan**: decomposed tasks with wave assignments and dependencies

## Writing Style

- Use precise, unambiguous language — avoid "should" and "might"
- Prefer concrete examples over abstract descriptions
- Each AC must be independently verifiable without human judgment
