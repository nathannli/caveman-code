# CaveKit Documentation Inversion

Write documentation from the user's perspective first, then derive implementation from what the documentation promises. This is the inverse of the typical code-then-document pattern.

## Approach

- Draft the README or API reference before writing any implementation
- The documentation becomes a de-facto acceptance criterion
- Implementation is done when the documented behavior is observable
- Keeps documentation accurate by making it the source of truth
