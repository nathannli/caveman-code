# CaveKit UI Craft

Guidelines for building high-quality user interfaces under CaveKit — covering visual fidelity, interaction design, and component architecture.

## Principles

- Components are specified with visual acceptance criteria (screenshot comparisons or pixel-level descriptions)
- Accessibility is a first-class AC, not an afterthought
- Reuse tokens from the design system before introducing new values
- Interactive states (hover, focus, disabled, loading) are explicitly spec'd and checked
- UI ACs are validated with real browser renders, not unit tests alone
